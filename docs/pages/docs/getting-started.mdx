# Getting Started

<Steps>

### Install the CLI

```sh
dart pub global activate jolt_cli
```

### Project Setup

<Tabs items={['Existing Project', 'New Project']} defaultIndex="0">

<Tabs.Tab>

From the root of your Flutter project, run the following command:

```sh
jolt init
```

This will create a Flutter package called **'ui'** within your project.

This package is where you should keep all your UI code, widgets, themes, typography, colors, etc.

When you initialise Jolt, a whole bunch of styling is setup for you. This code is now yours to own and modify as you see fit.

Any Jolt widgets that you install will be added to this package in the lib/widgets folder. You can also add your own custom widgets to the same folder.

### App Setup

To convert your project to a Jolt project, replace your existing MaterialApp or CupertinoApp with JoltApp.

```dart
import 'package:ui/ui.dart';

class App extends StatelessWidget {
  const App({super.key});

  @override
  Widget build(BuildContext context) {
    return JoltApp.router(
      themes: themes,
      styles: styles,
      routerConfig: AppRouter.instance.config(),
    );
  }
}
```

Make sure to replace your Material imports with your new ui package.

```dart
// import 'package:material/material.dart';
import 'package:ui/ui.dart';
```

To create a new Flutter project using the Jolt CLI, you can use the create command.

```sh
jolt create my_app
```

A new project will be created with the following optional features:

- Routing
- Localization
- State management with Riverpod
- Themes
